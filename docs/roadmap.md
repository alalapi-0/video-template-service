# 路线图（多轮演进）

以下为建议里程碑，**可随团队节奏调整**。Round 0 已完成骨架，不要自动跳入下一阶段而不评审需求。

| 轮次 | 主题 |
|------|------|
| **Round 0** | 项目骨架与文档（当前） |
| **Round 1** | 前端上传 UI 与后端上传字段、错误提示彻底打通 |
| **Round 2** | 模板系统完善（校验、版本号、预览示意图可选） |
| **Round 3** | 任务状态系统（持久化或多阶段：queued/processing/failed） |
| **Round 4** | 本地 FFmpeg **最小**合成（单主视频缩放铺满画布导出） |
| **Round 5** | 画中画模板合成 |
| **Round 6** | 文本叠加与标题排版（字体文件与路径约定） |
| **Round 7** | AI 文案从 Mock 切到「可配置真实 API」（仍保留降级 Mock） |
| **Round 8** | 多模板并行维护与示例素材包 |
| **Round 9** | 简单预览页面（服务端静态输出 + 前端 video 播放） |
| **Round 10** | MVP 闭环测试（端到端样片） |
| **Round 11** | 错误处理与磁盘清理策略 |
| **Round 12** | 部署准备（Docker、环境变量、FFmpeg 镜像） |
| **Round 13** | 用户体验优化（文案、 Loading、失败重试） |
| **Round 14** | 复盘与必要重构 |

# AI 使用规划

## AI 负责什么

- 将用户**自然语言**拆成结构化字段：标题、副标题、标签列表、适合字幕的短正文。
- 根据长度给出**字号建议**、是否换行、是否截断提示。
- 按 `template_id` 选择语气或长度上限（例如竖屏模板标题更短）。

## AI 不负责什么

- **不负责**视频合成、转场、调色、关键帧动画。
- **不负责**替代模板几何布局（矩形位置仍由模板 JSON 固定）。
- **不负责**无界创作（应限制 token 与输出格式，避免跑题）。

## 哪些地方用规则即可

- 字符数阈值、敏感词本地过滤、标签个数上限。
- 空文案、过短文案的默认占位字。
- 固定栏目名、品牌尾板文案（完全不必调用模型）。

## 如何控制成本

- 默认走**小模型或规则**；仅当用户勾选「智能标题」等再调用大模型。
- 缓存相同 `raw_text + template_id` 的结果一段时间。
- 限制输入长度与输出字段数量；JSON schema 约束返回。

## 如何避免过度依赖大模型

- 把 AI 当作「**文案助理**」，核心流水线仍由模板 + FFmpeg 驱动。
- 单元测试应覆盖「无 AI / Mock AI」路径，保证主流程可离线演示。

## Round 0

- `ai_text_service.py` 返回固定形态 Mock 数据，**无外部网络请求**。
